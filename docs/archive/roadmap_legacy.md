# Дорожная карта (локальная платформа курсов)

> **Архив**: документ сохранён для истории. С апреля 2025 года актуальная стратегия развития описана в `docs/planning/`.

Фокус на локальном использовании: курсы создаются один раз и редко редактируются; внешние интеграции, многоязычие и нотификации не требуются.

## Что уже есть
- Роли: студент, администратор, суперадмин; разграничение доступа.
- Каталог курсов с модулями (теория и квиз), учёт прогресса.
- Админ-инструменты: создание студентов, сброс паролей, просмотр прогресса.
- Суперадмин: создание курсов, модулей и вопросов.

## Недавно сделано
- Создание студента со статическим ID и автоматическим логином `id@сервер`, модальное окно с логином/паролем.
- Микро-навигация по модулю: хлебные крошки, прогресс «Шаг X из Y», ссылки на предыдущий/следующий модуль.
- Поиск студентов по статическому ID и фильтры по серверу на вкладке администратора.
- Повторные попытки квизов, карточка результата с кнопкой «Повторить попытку».
- История попыток модуля (лучший результат, последняя попытка, список попыток).
- Редактор теории с живым предпросмотром и Markdown-тулбаром для суперадмина.
- Система групп и тегов: курсы привязываются к группе, теги управляют доступом; администраторы назначают теги студентам.
- Справочник для групп и тегов в панели суперадмина (создание и список).
- Быстрый предпросмотр курса глазами студента: отдельная страница и ссылки из редактора, просмотр модулей в безопасном режиме.
- Фильтры по группе и тегу в списке курсов суперадмина.
- Быстрый мастер при создании материала (типовые модули) и дублирование материалов/модулей.

## Приоритетные доработки (фокус на ближайшее время)
- Улучшения UX квиза
  - Пояснения к ответам после сдачи; подсветка неверных.
  - Перемешивание вариантов ответов для всех вопросов.

## Средний приоритет (по мере необходимости)
- Черновики и «публикация» модулей; предпросмотр контента.
- Настройки курса: проходной балл, лимит попыток, кулдаун, включение/выключение таймера.
- Мини-резервные копии: кнопка «Скачать/Загрузить базу» (SQLite-файл) для локальных бэкапов.
- Мини‑аналитика в админке
  - Средний балл по курсу, число заваленных попыток, «топ‑вопросы», где чаще всего ошибаются.
  - Отчёт по группе/серверу (агрегация по выбранному серверу: Рублёвка/Арбат/Патрики/Тверской) с выгрузкой CSV.

## Низкий приоритет / позже
- Простые отчёты (выгрузка CSV по прогрессу группы/сервера).
- Минимальный журнал админ-изменений (кто и что изменил) — только при появлении потребности.

## Не планируем
- Многоязычие и система словарей (платформа локальная).
- Нотификации (e-mail, веб‑пуши) и внешние интеграции.
- Сложные CI/CD, облачные развёртывания.

## Технические заметки
- Минимальная валидация данных (уникальность логина, длины полей) чтобы избежать дубликатов и мусора.
- Конфигурация через .env, без внешних сервисов; фиксированные порты по умолчанию.
- Сервер — перечислимый список значений (Рублёвка, Арбат, Патрики, Тверской) на фронтенде (select) и в БД (CHECK/enum); валидация на API.
- Хранить отдельно `staticId` (вводимый админом) и `login` (формат `{staticId}@{server}`); индекс уникальности по `login`.
- Спецификация по группам и тегам: см. `docs/spec_groups_tags.md`.
