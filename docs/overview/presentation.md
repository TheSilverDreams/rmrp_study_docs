# Презентация: Материалы RMRP

## Назначение
Платформа «Материалы RMRP» — единая точка для хранения учебных материалов и контрольных листов, необходимых сотрудникам RMRP. Решает задачи:
- централизованное предоставление информации и инструкций;
- контроль знаний через квизы и фиксацию прогресса;
- упрощённое ведение пользователей и прав доступа без внешних сервисов.

## Кому полезна
- **Суперадминистратор**: формирует материалы, управляет структурами (группы, теги), отслеживает полный контент.
- **Администратор**: создаёт и сопровождает учетные записи, контролирует прохождение материалов, сообщает доступы сотрудникам.
- **Пользователь**: получает доступ к необходимым материалам, отмечает изучение теории и сдаёт контрольные тесты.

## Ключевые возможности
### Для суперадмина
- Управление материалами: создание, мастер быстрых модулей (теория + квиз), дублирование материалов/модулей, предпросмотр.
- Справочник групп и тегов: гибкая настройка структуры и доступа.
- Банк вопросов: поддержка одиночного и множественного выбора, теги и пояснения, авто-пул с ротацией.
- Настройки квиза: количество вопросов, проходной балл, фильтрация по тегам.

### Для администратора
- Управление учетными записями (ник, статик ID, сервер, группа, теги).
- Автоматическая генерация логина (`staticId@server`) и пароля при создании/сбросе.
- Отслеживание прогресса: модули, статус, группа материала, задействованные теги.

### Для пользователя
- Просмотр теоретических модулей с Markdown и встроенными видео.
- Кнопка «Я ознакомился ещё раз, я молодец» для повторного подтверждения.
- Квизы с перемешиванием вариантов и ротацией вопросов; после сдачи — подробный разбор с подсветкой правильных/неверных ответов и пояснениями.

## Технические особенности
- **Фронтенд**: React + Vite + TypeScript + TailwindCSS;
- **Бэкенд**: Express + TypeScript, база SQLite (better-sqlite3);
- **Роли**: пользователь, администратор, суперадмин;
- **Ротация вопросов**: на каждой попытке выбираются наименее показанные пользователю вопросы (по `questionCount`), ответы перемешиваются;
- **Дублирование**: материалы и модули можно копировать, сохраняя структуру и вопросы.

## Сценарий использования
1. **Суперадмин** заводит группы, теги, материалы через мастер и наполняет их модулями/вопросами.
2. **Администратор** создаёт учетные записи, назначает группы/теги, контролирует прохождение.
3. **Пользователь** изучает материалы, отмечает теорию, проходит квизы. Система фиксирует попытки и показывает отчёт.

## Планы развития (ключевые)
- Улучшение UX квиза: вывод пояснений и подсветки (уже частично реализовано, останется fine-tuning).
- Дополнительная аналитика (успешность/«узкие места»).
- Импорт/экспорт материалов и вопросов.

Платформа готова к локальному развёртыванию, покрывает базовые потребности по обучению и контролю знаний внутри компании и может быть расширена под новые процессы.
