# План развития портала RMRP: обзор

Документ описывает стратегию эволюции платформы и увязывает её с пятью крупными этапами. Каждый этап имеет собственный файл с детальным планом:
- [Этап A — Завершение блока курсов](stage-a-course-readiness.md)
- [Этап B — Внутренний портал документации](stage-b-internal-docs.md)
- [Этап C — Управление администрацией и разрешениями](stage-c-admin-access.md)
- [Этап D — Безопасность, сессии и мастер-панель суперадмина](stage-d-security-ops.md)
- [Этап E — Корпоративный портал и аналитика](stage-e-corporate-portal.md)

## Принципы
- **Поэтапность**: переходим к следующему блоку только после «definition of done» для текущего.
- **Прозрачность**: фиксируем владельцев задач, ключевые риски, контрольные точки.
- **Безопасность по умолчанию**: привилегированные функции и данные защищаются с самого начала.
- **Расширяемость**: решения, принятые на ранних этапах, закладываются с учётом последующих расширений (RBAC, аудит, аналитика).

## Критерии готовности этапа
- Все заявленные фичи реализованы и покрыты проверками (юнит + регрессионные сценарии, smoke-тесты).
- Обновлена пользовательская и техническая документация.
- Проведено ревью безопасности и проведена демо-сессия для владельцев процесса.
- Подготовлен отчёт о выполненных задачах и «lessons learned» для следующего этапа.
