# План: аудит логов и административный интерфейс

## Цель
Повысить прозрачность действий администраторов и студентов, облегчить расследование инцидентов, упростить работу супер-администраторов.

## Ближайшие задачи
- Логировать IP-адрес и User-Agent при каждом входе в систему (сохранение в базе).
- Добавить внутреннюю страницу с агрегированными логами авторизаций (доступна только администраторам).
- Расширить карточку пользователя в админке: отобразить историю входов (IP, дата/время, устройство) и вынести основные действия (сброс пароля, удаление, блокировка).
- Переработать текущую страницу «Прогресс»: перенести её на уровень ниже, чтобы на основном экране профиля администратора показывалась ключевая информация и действия.

## Открытые вопросы
- Фильтры/поиск по логам (по времени, IP, событиям) — не нужны на первом этапе.
- Уведомления при подозрительных входах (география, устройство) — не требуются.
- Хранение логов — 1 месяц, маскирование IP не требуется.
- Настроить права доступа: просмотр логов только для сотрудников с `staff.manage`.
